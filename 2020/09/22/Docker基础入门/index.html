Docker基础入门

发表于 2020-09-22 | 分类于 Docker

Docker是什么?

Docker是一个开源的应用容器引擎,让开发者可以打包他们的应用以及依赖包到一个可移植的镜像中,然后发布到任何流行的Linux或Windows机器上,也可以实现虚拟化。容器是完全使用沙箱机制,相互之间不会有接口。总体来说,Docker的接口非常简单,用户可以方便地创建和使用容器,把自己的应用放入容器.容器还可以进行版本控制,复制和修改,就像管理普通的代码一样。

为什么会出现Docker?

环境配置

软件开发最大的麻烦事之一就是环境配置.用户计算机的环境都不相同,你怎么知道自己的软件能在哪些机器跑起来?用户必须保证两件事:操作系统的设置,各种组件的安装,只有他们正确软件才能运行.如果某些老旧的模块和当前环境不兼容,那就麻烦了。开发者常常说:’他在我的机器上好好的’.环境配置如此麻烦,换一台机器就要重新来一次,这样不能从根本上解决问题,能不能从根本上解决问题,软件可以带环境安装?也就是说,安装的时候,把原始环境一摸一样的复制过来。

虚拟机

虚拟机就是带环境安装的一种方案.他可以在一种操作系统里面运行另一种操作系统.比如在Windows系统运行Linux系统。因为虚拟机看上去和真实系统一摸一样,对于底层系统来说,虚拟机就是一个普通文件,不需要就删掉,对其他部分不影响。
虽然用户可以通过虚拟机还原软件的原始环境,但是这个方法有几个缺点:
1.资源占用多
虚拟机会单独占一部分内存和硬盘空间.他运行的时候,其他程序就不能使用这些资源了。哪怕虚拟机里面的应用程序,真正使用的内存只有1MB,虚拟机依然需要几百MB的内存才能运行。
2.冗余步骤多
虚拟机是完整的操作系统,一些系统级别的操作步骤,往往无法跳过,比如用户登录
3.启动慢
启动操作系统需要多久,启动虚拟机就要多久

Docker和虚拟机的不同

传统虚拟机,虚拟出一个硬件,运行一个完整的操作系统,然后在这个系统上安装和运行软件;Docker容器内的应用直接运行在宿主机的内核中,容器是没有自己的内核的,也没有虚拟硬件,所以很轻便,每个容器之间都是隔离的

Docker组成

镜像(image):docker镜像就好比是一个模版,可以通过这个模版来创建容器服务;可以通过这个镜像可以创建多个容器
容器(container):Docker利用容器技术独立运行一个或者一组应用,通过镜像来创建(最终服务运行或者项目都是在容器中),可以看作是一个简易的linux系统
仓库(repository):用来存放镜像的地方
仓库分为公有仓库和私有仓库

常用命令

帮助命令

  • docker version 显示docker的版本信息

  • docker info 显示docker的系统信息

  • docker –helo 帮助命令

镜像命令

docker images:查看所有本地主机上的镜像
参数: -a :列出所有镜像 -q :只显示镜像id
docker search 镜像名称:搜索镜像
常用参数: --filter 搜索stars大于3000的mysql镜像
docker pull 镜像名称:拉取镜像
docker rmi 【-f(强制删除)】 镜像id
批量全部: 批量删除:docker rmi -f 容器id 容器id。。

容器命令(有了镜像才可以创建容器)

docker run [可选参数] 镜像名称

先拉取一个centos镜像

参数说明:

  • –name 给运行的容器取名

  • -d 后台方式运行

    问题:docker ps,发现容器停止运行了 docker使用后台运行,就必须要有一个前台进程;如果没有docker会发现没有应用,就会自动停止

  • -it 使用交互方式运行,进入容器内部

  • -p 指定端口映射8081(宿主机端口):8080(容器内部端口)

  • P 随机分配一个端口

docker ps:列出正在运行的容器
参数: -a 列出所有容器 -n=? 显示最近创建的容器 -q 只显示容器id
退出容器
  • exit:容器停止并退出
  • 快捷键:ctrl+P+Q
删除容器 docker rm 容器id(不能删除正在运行的容器,除非使用rm -f强制删除)
全部删除:
启动和停止容器

启动容器:docker start 容器id

停止容器:docker stop 容器id

重启容器:docker restart 容器id

强制停止:docker kill 容器id

docker logs 容器id(查看容器日志)

参数:
-tf : 显示日志

–tail number : 显示日志行数

查看容器中进程信息 docker top 容器id(容器必须启动状态)
查看镜像元数据 docker inspect 容器id
进入当前正在运行的容器

  • docker exec -it 容器id /bin/bash

  • docker attach 容器id

attach和exec区别就是exec会重新开启一个新的终端;attach不会开启新的终端
从容器内部拷贝文件到宿主机上 docker cp 容器id:容器内路径 宿主机路径
