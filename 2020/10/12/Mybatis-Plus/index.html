Mybatis Plus

发表于 2020-10-12 | 分类于 Mybatis Plus

简介

MyBatis-Plus简称MP)是一个MyBatis的增强工具,国产的开源框架,基于MyBatis,在MyBatis的基础上只做增强不做改变,为简化开发,提高效率而生。

特性

  • 无侵入:只做增强不做改变,引入它不会对现有工程产生影响
  • 损耗小:启动即会自动注入基本 CURD，性能基本无损耗，直接面向对象操作
  • 强大的 CRUD 操作：内置通用 Mapper、通用 Service，仅仅通过少量配置即可实现单表大部分 CRUD 操作，更有强大的条件构造器，满足各类使用需求
  • 支持 Lambda 形式调用：通过 Lambda 表达式，方便的编写各类查询条件，无需再担心字段写错
  • 支持主键自动生成：支持多达 4 种主键策略（内含分布式唯一 ID 生成器 - Sequence），可自由配置，完美解决主键问题
  • 支持 ActiveRecord 模式：支持 ActiveRecord 形式调用，实体类只需继承 Model 类即可进行强大的 CRUD 操作
  • 支持自定义全局通用操作：支持全局通用方法注入（ Write once, use anywhere ）
  • 内置代码生成器：采用代码或者 Maven 插件可快速生成 Mapper 、 Model 、 Service 、 Controller 层代码，支持模板引擎，更有超多自定义配置等您来使用
  • 内置分页插件：基于 MyBatis 物理分页，开发者无需关心具体操作，配置好插件之后，写分页等同于普通 List 查询
  • 分页插件支持多种数据库：支持 MySQL、MariaDB、Oracle、DB2、H2、HSQL、SQLite、Postgre、SQLServer 等多种数据库
  • 内置性能分析插件：可输出 Sql 语句以及其执行时间，建议开发测试时启用该功能，能快速揪出慢查询
  • 内置全局拦截插件：提供全表 delete 、 update 操作智能分析阻断，也可自定义拦截规则，预防误操作

常用注解

@TableName:映射数据库的表名

刚才不知道有没有一个疑问,为什么他会去查user这张表呢?因为我们的实体类是user,所以mybatis plus自动会根据实体类去数据库查询,那这时候我们把实体类名称换掉一个数据库不存在的,就会报错。那么怎么办呢?我们就可以使用@TableName这个注解,和数据库对应

这就表示我们查询user1这张表的数据就去查询user这张表。

@TableId:设置主键映射(主键要使用包装类型)

value:映射主键字段名
type:设置主键类型,主键的生成策略,默认雪花算法实现

AUTO:数据库自增
测试数据库结果 他会根据数据库中最后一行记录实现id自增长
NONE:雪花算法实现(默认)
测试 结果 使用雪花算法的时候,使用Integer类型的id,不能够存储,这时候我们需要将Integer转为Long类型 结果 数据库中
INPUT:需要开发者手动赋值
测试:我们不手动赋值,他也会根据数据库中最后一个字段自增 结果
ASSIGN_ID:Mybatis Plus分配id,可以使用Long,Integer,String类型

@Version:标记乐观锁,通过version字段来保证数据的安全性,当修改数据的时候,会以version作为条件,当条件成立的时候才会修改成功。

假如数据中有一个version字段,version=1,每次执行完毕都会update。。set version=2 where version=1,将version字段值+1;那如何保证数据安全呢?假如有两个线程同时进来
第一个线程:update…set version=2 where version=1
第二个线程:update…set version=2 where version=1
当第一个线程修改后,version字段变为2了,第二个线程就不能再更改了,这样就可以保证数据的安全性
