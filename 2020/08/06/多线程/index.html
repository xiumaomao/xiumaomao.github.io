多线程

发表于 2020-08-06 | 分类于 多线程

线程和进程

进程是执行程序的一次过程,它是一个动态的概念。是系统资源分配的单位。
线程是依赖进程的,线程是cpu执行的最小单位.一个进程可以包括多个线程。两者的关系举个例子
我们在看电影的时候,你qq也是开着的,电影和qq就是两个进程.电影里的声音和图像就是两个线程分别控制。

创建线程的方式

1.继承Thread,重写run方法,然后在调用start方法开启线程。
public class Main extends Thread{
public void run(){
for(int i=0;i<10;i++){
System.out.print(i);
}
}
public static void main(String[] args) {
Main main=new Main();
main.start();
}
}
代码结果0123456789

注意:start方法是用来开启现成的方法,run方法只是一个普通调用的方法

例如:
public void run(){
for(int i=0;i<10;i++){
System.out.print(i);
}
}
public static void main(String[] args) {
Main main=new Main();
main.start();
for(int i=0;i<10;i++){
System.out.print(“我是主线程”+i);
}
}

2.实现Runnable接口,重写run方法,将线程作为参数传递
public class Main implements Runnable{
public void run(){
for(int i=0;i<10;i++){
System.out.print(i);
}
}
public static void main(String[] args) {
Main main=new Main();
new Thread(main).start();
for(int i=0;i<10;i++){
System.out.print(“我是主线程”+i);
}
}
}

线程的状态

1.新建状态(new) 创建后未启动的线程的状态 new一个线程,未调用start方法
2.就绪状态:当调用start()方法之后,线程立即进入就绪状态
3.运行状态:cpu调度线程.
4.阻塞状态:当调用sleep,wait同步方法时,线程进入阻塞状态,阻塞结束后,继续等待cpu调度。
5.终止状态:线程已经结束运行

如何让线程终止?

其实jdk提供了stop()和destory()方法终止线程,但是已经被废弃,现在不推荐使用。我们可以给自己设置一个标识位,如下面代码:
public class Main {
public static void main(String[] args) {
Thread1 thread1=new Thread1();
Thread thread=new Thread(thread1);
thread.start();
for(int i=0;i<10;i++){
if(i==8){
thread1.stop();
}
}
}
}
class Thread1 implements Runnable{
private Boolean flag=true;
@Override
public void run() {
while (flag){
for(int i=0;i<10;i++){
System.out.println(i);
}
}
}
public void stop(){
this.flag=false;
}
}

sleep方法和wait方法

相同点:都可以暂停线程的执行,使线程进入等待状态。
不同点:1.sleep没有释放锁,wait会释放锁。2.sleep是Threa类的静态方法,wait是Object对象的实例方法 3.sleep可以在任何方法中使用,wait只能在synchronized方法中使用。

实现runnable接口和实现callable接口创建线程的区别？

创建线程的方式大家都知道继承Thread类,重写run方法;实现runnable接口,重写run方法;实现callable接口,重写call方法。用的最多的就是前两种,那么继承Threa类和实现runnable方法哪个好呢?以我个人而言,我觉得实现runable接口更好一些,因为接口可以多实现,抽象类不能多继承,扩展性好一些.实现runable接口和实现callable接口区别？
1.实现runnable方法没有返回值,实现callable接口有返回值
2.实现runnable方法不会抛出异常,实现callable接口会抛异常。

线程池

线程池其实和我们之间学过的jdbc连接池差不多是一个概念。线程池中存放了一些创建好的线程,等系统需要的时候直接调用即可

线程池优点？

有时候系统需要在短时间内处理很多请求,如果每次请求都需要创建一个线程池的话,系统就需要不断的进行线程的创建和销毁,消耗时间和性能,使用线程池可以重用线程池中的线程,减少系统开销,节省时间和性能。

线程池创建？

在java中创建线程池常用的类是ThreadPoolExecutor,该类的全参构造如下:
public ThreadPoolExecutor(int corePoolSize,
int maximumPoolSize,
long keepAliveTime,
TimeUnit unit,
BlockingQueue workQueue,
ThreadFactory threadFactory,
RejectdExecutionHandlle handle)
参数介绍:
1.corePoolSize:线程池中核心线程池的数量
2.maximumPoolSize:线程池中能拥有的最大线程数
3.workQueue:阻塞队列
4.keepAliveTime:表示空闲线程的存活时间
5.handle:表示当workQueue已满的时候,且线程中的数量最大值的时候,线程池拒绝新任务时采取的策略。一般有四种
(1).ThreadPoolExecutor.AbortPolicy():抛出RejectdExecutionException异常
(2).ThreadPoolExecutor.CallerRunsPolicy:由向线程中提交任务的线程来执行该任务
(3).ThreadPoolExecutor.DiscardOldestPolicy:抛弃最旧的任务
(4).ThreadPoolExecutor.DiscadPolicy:抛弃当前任务
6.threadFactory:指定创建线程的工厂

4种常见的线程池:
1.Exectors.newCacheThreadPool() 可缓存线程池。 先查看线程池中有没有以前建立的线程,如果有,就直接使用。如果没有就创建一个新的线程加入到线程池中。
示例代码:
ExecutorService cachedThreadPool = Executors.newCachedThreadPool();
2.Exectors.newFixedThreadPool() 可重用固定个数的线程池
示例代码:
ExecutorService fixedThreadPool = Executors.newFixedThreadPool(3);
3.Exectors.newScheduledThreadPool() 定长线程池 支持定时及周期性任务执行。
ScheduledExecutorService scheduledThreadPool = Executors.newScheduledThreadPool(5);
4.Exectors.newSingledThreadPool() 创建一个单线程的线程池,只会创建一个线程来执行任务,如果这个线程异常结束，那么会在重新创建一个线程，他会始终保持线程池里只有一个线程。这样可以保持任务在队列中是按顺序执行的。

线程池的流程

1.当线程池中线程数小于corePoolSize时,新增任务将创建一个新线程池中存在空闲线程。
2.当线程池中线程数达到corePoolSize时,新增任务将放入到workQueue队列中,等待线程中任务调度。
3.当workQueue已满,并且maximumPoolSize > corePoolSize时,新提交任务会创建新线程执行任务。
4.当workQueue已满,且提交任务数超过maximumPoolSize,任务由RejextExecutionHandler处理。
