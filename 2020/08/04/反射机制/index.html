反射机制

发表于 2020-08-04 | 分类于 反射

什么是反射机制?

Java反射机制是在运行状态时,对于任意一个类,都能够直到这个类的所有属性和方法;对于任意一个对象,都能够调用它的任意一个方法和属性;这种动态获取的信息以及动态调用对象的方法的功能称为Java语言的反射机制。要想剖解一个类必须先获取该类的字节码文件对象。而剖解使用的就是Class类中的方法,所以先要获取到每一个字节码文件对应的Class类型的对象。

总结:反射就是Java类中各个成分映射成一个个Java对象

对于反射的理解

既然有了反射,那’正’的是什么呢？一般来说,使用某个类的时候,我们一般都会去看看类中有哪些变量,有哪些方法,他是用来做什么的,方法参数是什么.以便与我们今后调用它。就例如:Person p=new Person(); p.run();我们线new一个类的实例,再去调用里面的方法。这就是‘正’
那反射就是一开始我们并不知道初始化的类对象是什么样子的,自然我们也不可以new对象实例去操作他,这时我们可以使用反射去操作这个类.

以上代码可以直接运行吗,很显然不可以,我们需要通过一个命令javac编译,在编译的过程中会检查,如果代码语法没有问题,会在硬盘上形成一个字节码文件Person.class,字节码文件里放的就是这个类的一些信息,主要有成员变量,构造方法,成员方法。这个阶段我们称之为源代码阶段。(这个阶段代码还存在硬盘上)
2.在硬盘上形成一个字节码文件后,java中通过类加载器ClassLoader将字节码文件加载到内存中,在内存中表述这个字节码文件称之为类对象,类对象中放的是字节码中共同属性,例如成员变量,构造方法,成员方法。这三个部分java把它封装成不同的对象。成员变量封装为field对象,构造方法封装为constructor对象,成员方法封装为method对象.这个阶段我们称之为类对象阶段。
3.创建对象Person p=new Person(); 这个阶段new出来的对象都放在内存的堆中,这个阶段成为运行时阶段。

封装的好处

1.在运行过程中可以操作这些对象(例如我们平时操作字符串的时候,通过字符串.的方式会出来很多方法,这其实就用到了反射机制)
2.可以解藕。

3.对象.getClass,通过对象的getClass方法

反射的应用

JDBC操作数据库第一步加载数据库驱动,Class.forName(“com.mysql.jdbc.Driver”);这里是 MySQL 数据库，假如某一天我们想换成 Oracle数据库,你可能会修改forName()方法中的参数为 Oracle 数据库驱动名。但其实我们还可以在不修改代码的情况下实现修改，就像大多数框架那样，我们可以使用个写个配置文件,forName()方法中的参数在配置文件中动态读取，编译好的代码完全不用动，这里其实就是反射的一个应用。另外在我们写代码时，在对象后面敲一个 . ，IDE（如 Eclipse） 就会自动帮我们列出该对象有的方法，这里其实就是IDE使用了反射，通过对象找到该类对应的 Class 对象，从而就可以找到类中的属性和方法。

总结

反射可以是我们的代码更具灵活性,但是反射会更消耗资源,如果不需要动态创建对象,那么就不需要用到反射。
