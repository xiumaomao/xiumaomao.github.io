反射机制

发表于 2020-08-04 | 分类于 反射

什么是反射机制?

Java的反射是指在程序运行状态中,可以任意了解一个类的成员变量和方法,对于任意一个对象可以调用他的属性和方法.这种动态获取程序信息以及动态调用对象的功能称为Java语言的反射机制。以上就是百度的原话,在我看来什么是反射呢，它其实就是将类的各个组成部分封装成了其他对象。比如说将类中的成员变量封装成了field对象,构造方法封装成constructor对象,成员方法封装成method.接下来我们往下看。

对于反射的理解

既然有了反射,那’正’的是什么呢？一般来说,使用某个类的时候,我们一般都会去看看类中有哪些变量,有哪些方法,他是用来做什么的,方法参数是什么.以便与我们今后调用它。就例如:Person p=new Person(); p.run();我们线new一个类的实例,再去调用里面的方法。这就是‘正’
那反射就是一开始我们并不知道初始化的类对象是什么样子的,自然我们也不可以new对象实例去操作他,这时我们可以使用反射去操作这个类.
举一个代码例子
public class Phone {
private int price;
public int getPrice() {
return price;
}
public void setPrice(int price) {
this.price = price;
}

public static void main(String[] args) throws Exception{
    //正常的调用
    Phone phone = new Phone();
    phone.setPrice(5000);
    System.out.println("Phone Price:" + phone.getPrice());
    //使用反射调用
    Class clz = Class.forName("com.xxp.api.Phone");
    Method setPriceMethod = clz.getMethod("setPrice", int.class);
    Constructor phoneConstructor = clz.getConstructor();
    Object phoneObj = phoneConstructor.newInstance();
    setPriceMethod.invoke(phoneObj, 6000);
    Method getPriceMethod = clz.getMethod("getPrice");
    System.out.println("Phone Price:" + getPriceMethod.invoke(phoneObj));
}

}
这个例子很好的诠释了‘正’和’反’,正就是操作之前确定类对象的结构通过new的方式去获取类对象,反射是在操作前不知道类对象是什么样子的.我们可以在程序运行过程中对类对象操作。

我们先来了解一下java程序在计算机中经历的阶段

1.首先我们定一个类文件，例如Person.java,如下面代码
public class Person {

private int id;
private String name;

//空参构造器
public Person() {
}

public void run(){
    System.out.println("人会跑步");
}

}
以上代码可以直接运行吗,很显然不可以,我们需要通过一个命令javac编译,在编译的过程中会检查,如果代码语法没有问题,会在硬盘上形成一个字节码文件Person.class,字节码文件里放的就是这个类的一些信息,主要有成员变量,构造方法,成员方法。这个阶段我们称之为源代码阶段。(这个阶段代码还存在硬盘上)
2.在硬盘上形成一个字节码文件后,java中通过类加载器ClassLoader将字节码文件加载到内存中,在内存中表述这个字节码文件称之为类对象,类对象中放的是字节码中共同属性,例如成员变量,构造方法,成员方法。这三个部分java把它封装成不同的对象。成员变量封装为field对象,构造方法封装为constructor对象,成员方法封装为method对象.这个阶段我们称之为类对象阶段。
3.创建对象Person p=new Person(); 这个阶段new出来的对象都放在内存的堆中,这个阶段成为运行时阶段。

封装的好处

1.在运行过程中可以操作这些对象(例如我们平时操作字符串的时候,通过字符串.的方式会出来很多方法,这其实就用到了反射机制)
2.可以解藕。

class类对象获取方式

1.class.forName(“类的全限定类名”)
2.类.class,通过类名的属性获取
3.对象.getClass,通过对象的getClass方法
Class clazz=Class.forName(“com.company.entity.Person”);
System.out.println(clazz);

Class clazz1= Person.class;
System.out.println(clazz1);

Person p=new Person();
Class clazz2=p.getClass();
System.out.println(clazz2);

System.out.println(clazz==clazz1);
System.out.println(clazz==clazz2);

运行结果:
class com.company.entity.Person
class com.company.entity.Person
class com.company.entity.Person
true
true
因此我们可以知道一个字节码文件在一次运行程序过程中只会加载一次,不管你通过哪种方式。

class类对象的功能

1.获取成员变量
(1).Field [] getFields 获取所有public修饰的成员变量

private int id;
private String name;


  Class clazz= Person.class;
  Field[] f=clazz.getFields();
  for(Field field:f){
      System.out.println(field);
  }
}

打印结果没空.getFields()方法得到的成员变量都是public修饰的
(2).Field getField(String name)：得到指定名称的public修饰的变量

(3).Field [] getDeclaredFirlds() 获取所有的成员变量(不考虑修饰符的限制)
(4).Field getDeclaredField() 获取指定名称的成员变量
当我们获取单个private修饰的变量时,程序会报错。这时我们只要加一行代码
Field o1= (Field) f2.get(“id”);
o1.setAccessible(true);就可以了

2.获取成员方法
(1)Method getMethod(String 方法名称) 空参方法
Class clazz= Person.class;
Method m=clazz.getMethod(“run”);
Person p=new Person();
m.invoke(p);
结果如下:
人会跑步
Method getMethod(String 方法名称) 有参方法
Person:
public void run(int id){
System.out.println(id+”在跑步”);
}

Class clazz= Person.class;
Method m=clazz.getMethod("run",int.class);
Person p=new Person();
m.invoke(p,1);

其他方法类似

3.获取构造方法
(1).Constructor getConstructor(.class)
Class clazz= Person.class;
Constructor constructor=clazz.getConstructor(int.class,String.class);
System.out.println(constructor);
结果如下
public com.company.entity.Person(int,java.lang.String)
构造方法主要用来创建对象,里面又一个newInstance方法,如下图
Class clazz= Person.class;
Constructor constructor=clazz.getConstructor(int.class,String.class);
Object person= constructor.newInstance(23,”maomao”);
System.out.println(person);
结果如下:
Person{id=23, name=’maomao’}
注意:如果是空参构造,可以直接使用class中的newInstance方法更简便。
(2).Constructor getDeclaredConstructor()使用方法和上述一样

反射的应用

JDBC操作数据库第一步加载数据库驱动,Class.forName(“com.mysql.jdbc.Driver”);这里是 MySQL 数据库，假如某一天我们想换成 Oracle数据库,你可能会修改forName()方法中的参数为 Oracle 数据库驱动名。但其实我们还可以在不修改代码的情况下实现修改，就像大多数框架那样，我们可以使用个写个配置文件,forName()方法中的参数在配置文件中动态读取，编译好的代码完全不用动，这里其实就是反射的一个应用。另外在我们写代码时，在对象后面敲一个 . ，IDE（如 Eclipse） 就会自动帮我们列出该对象有的方法，这里其实就是IDE使用了反射，通过对象找到该类对应的 Class 对象，从而就可以找到类中的属性和方法。

总结

反射可以是我们的代码更具灵活性,但是反射会更消耗资源,如果不需要动态创建对象,那么就不需要用到反射。
