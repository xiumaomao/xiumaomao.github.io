什么是Java？

Java是SunMicrosystems于1995年首次发布提出的一种编程语言和计算平台.计算平台就是在电脑中运行应用程序的环境,包括硬件环境和软件环境。
Java是快速的,安全和可靠的。Java主要分为三个版本

  • JavaSE(J2SE)(Java2PlatformStandardEdition,java平台标准版)
  • JavaEE(J2EE)(Java2Platform,EnterpriseEdition,java平台企业版)
  • JavaME(J2ME)(Java2PlatformMicroEditio,java平台微型版)

Java特点

  • java是一门面向对象的编程语言
    什么是面向对象？面向对象(Object Oriented)是一种软件开发思想。他是对现实世界的一种抽象,面向对象会把相关的数据和方法组织为一个整体来对待。
    相对的另外一种开发思想就是面向过程的开发思想,什么是面向过程？面向过程(Procedure Oriented)是一种以过程为中心的编程思想。
    举一个例子很好的诠释了面向过程和面向对象的区别。把大象装进冰箱分几步。以面向过程的思想来看分三步.第一步把冰箱门打开,第二步把大象装进冰箱,第三步把冰箱们关起来。而以面向对象的思想来看首先我们需要创建大象这么一个对象，然后告诉大象进冰箱，至于他怎么进冰箱我们就不需要管了。
    面向对象:
    优点:扩展性高 维护成本低 缺点:上手难度比较高
    面向过程:
    优点：简单 缺点:如果代码量大维护性很差。
  • Java摒弃了C++中难以理解的多继承,指针,内存管理的概念
  • Java语言具有功能强大和简单易用两个特征,现在企业级开发,快速敏捷开发使Java成为越来越火的一门语言。
  • Java是一门静态语言,静态语言指的是在编译期间就能够知道数据类型的语言,在运行前就能够检查类型的正确性,一旦确定后就不能再更改。
  • Java具有平台独立性和可移植性。
    Java有一句非常大的口号:Write once,run anywhere也就是一次编写,到处运行.为什么Java能够吹出这么牛的口号,核心就是JVM,关于JVM我们单独来讲。
  • Java具有健壮性
    Java的强类型机制,异常处理,垃圾的自动收集等是Java健壮性的保证

下面我们来讨论面向对象的思想,面向对象的思想已经逐步取代了面向过程,Java是面向对象的高级编程语言,面向对象有如下特征:

  • 面向对象是一种常见的思想,比较符合人们的思考习惯;

类也是一种对象

我们如果需要造汽车,我们首先得画个图纸,图纸里需要有汽车的属性信息,颜色啊,座位啊等属性还有车的动作,然后我们需要拿着图纸去工厂造汽车。在面向对象的世界里,类就是图纸,汽车就是对象.属性就是这一类事物拥有的共同属性,动作就是这一类事物的功能。
总结:类描述了一类对象的行为和状态，而对象就是一个类的具体实例。

方法重载

在Java中一个很重要的概念就是方法的重载,他是类名的不同表现形式,其实构造函数也是重载的一种,另一种就是方法的重载。


如上面所示,就有两种重载的方式,一种是构造函数的重载,另一种就是方法的重载,重载的条件:

  • 方法名称必须相同
  • 参数类型必须不同(个数不同,或类型不同,参数类型顺序不同等)
  • 方法的返回类型可以相同也可以不同
  • 仅仅返回类型不同不足以成为方法的重载

方法的重写

方法的重载与重写虽然名字很相似,但却完全是不同的东西.方法重写的描述是对子类和父类之间的,而重载是同一类中的

重写的原则:

  • 重写的方法必须与父类保持一致,包括返回值类型,方法名,参数列表也都一样
  • 重写的方法可以用@Override注解来标识.
  • 子类中重写方法的访问权限不能低于父类中方法的访问权限。

面向对象特征

  • 封装
    封装是面向对象的核心思想，将描述某种实体的数据和这些数据的操作放在一起，形成一个封装体。他保证了类内部结构的完整性，是其他用户无法直接操作内部数据，提高了安全性。我们常见的就是定义一个类，把属性修改为private不让外界访问，然后通过自己定义的getter/setter方法去访问。如下图:
  1. 封装的好处: 只能通过内部规定的方法访问数据
  2. 隐藏内部实现细节
  • 继承
    继承就是子类继承父类的特征和行为，使得子类具有父类的方法和属性。或子类继承父类方法，使得子类具有父类相同的行为。
    特征:1.父类更通用一些，子类更具体一些。子类除了具有父类的特征和行为还有自己特殊的特征和行为。2.继承可以实现代码复用。3.继承使用关键子extends来实现。
  • 多态
    多态就是一种事物的多种表现形式。多态的前提是有继承或者实现，通过父类new一个子类的方式可以使用子类的方法。
    多态的前提是必须有继承或者实现.例如定义一个Person类,写一个student类继承person类，我们就可以使用Person p=new Student();这就是典型的多态，可以通过父类直接new子类的方式使用子类中的属性和方法。多态性使得能够利用同一类对象(父类)引用不同类的对象,以及根据所引用对象的不同,以不同的方式执行相同的操作。多态实现包括两种方式:重载和重写
    例如Animal a=new Cat();父类引用指向子类对象,Animal类中有一个eat方法,而Cat类继承Animal类,如果子类重写了父类的eat()方法,则调用的时候,就可以按照子类的形式调用,本质上是父类的方法,但是子类重写后就变成了另一种方式,这就是多态。

向上转型

向上转型代表了父类与子类之间的关系,其实父类和子类之间不仅仅有想上转型,还有向下转型

  • 向上转型:通过子类对象(范围小的)转化为父类对象(大范围),这种转换是自动完成的,不用强制。
  • 向下转型:通过父类对象(范围大的)转化为子类对象(小范围),这种转换不是自动完成的,需要强制指定。

static

static是Java中的关键字,它的意思是静态的,static可以用来修饰成员变量和方法。

  • static修饰的成员变量为静态成员变量,也称为类变量,类变量的生命周期和类相同,在整个应用程序执行期间都有效。
  • static修饰的方法成为静态方法,静态方法可以通过类名.方法调用
  • static除了可以修饰属性和方法之外,还有静态代码块的功能,可用于类的初始化操作。

final,finally,finalize

final:

  • final修饰类时,表明这个类不能被继承,
  • final修饰方法时,表明这个方法不能被任何子类重写
  • final修饰变量分为两种,一种是基本数据类型,表示数据类型的值不能被修改;一种是修饰引用类型,表示初始化后不能再让其指向另1个对象。
    finally是java异常处理中的,用来无论是否发生异常都会执行的一个操作,如果抛出异常，进入try-catch代码块中，最终都会执行finally里面的语句，所以一般都是用来进行数据库关闭，流关闭等操作。
    finalize是Object类定义的一个方法，他是在垃圾回收前执行的。

    抽象类和接口

    相同点:

1.都不能直接实例化,需要继承或者实现才可以重写父类的方法
2.派生类必须实现未实现的方法
不同点:
1.抽象类用abstract关键字修饰,接口使用interface关键字修饰
2.抽象类里都是抽象方法，而接口有抽象方法也有普通方法
3.抽象类不可以多继承，接口可以多实现
4.抽象类和接口的设计理念不同,抽象类体现的是‘is-a’的关系,而接口体现的‘like-a’的关系

那么问题来了,java既然有了基本数据类型为什么还需要包装类呢？
众所周知，java是面向对象的,但并不是”纯面向对象的”，例如我们经常使用的基本数据类型就不是对象，在世纪开发中我们需要将基本数据
类型转换成对象，例如:将基本数据类型存储到数组或者集合中。为了解决这个问题，java在每个基本数据类型的基础上又引入了包装类。

装箱与拆箱

装箱简单来说就是自动将基本数据类型转换为包装类型；拆箱就是将包装类型转换为基本数据类型。
我们来看看下面代码:
Integer a=10; int b=a;
当执行Integer a=10的时候,系统为我们执行了Integer a=Integer.valueOf(10);
当执行int b=a的时候，系统为我们自动执行了a.intValue();
这是个典型的装箱与拆箱的过程,首先将a基本数据类型包装成Integer包装类,最后再将包装类转换成基本数据类型。
