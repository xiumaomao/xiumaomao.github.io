发表于 2020-08-11 | 分类于 springmvc框架

Srping MVC概述

说起Spring MVC,我们先来了解一下传统的MVC设计模式开发。用户发送请求—–>controller控制器—->java bean——>DB——>返回给用户
M代表的是model也就是java bean;V代表的是view视图,也就是传统的jsp,展示给用户的;C代表的是controller控制器,接收用户请求.SpringMvc是当前最主流的基于MVC设计模式开发的框架,是Spring框架的一个分支。他以Spring Ioc容器为基础,并利用容器的特性来简化它的配置。SpringMvc可以和Spring很好的结合起来开发,是java web开发者必须要掌握的框架。

核心组件

1.DispatcherServlet:前置控制器,是整个流程控制的核心,控制其他组件的执行,进行统一调度,降低组件之间的耦合性,相当于总指挥。
2.Handler:处理器,完成具体的业务逻辑,相当于Servlet或者Action
3.HandlerMapping: 处理器映射器 DispatcherServlet接收到请求后,通过HandlerMappinging将不同的请求映射到不同的Handler
4.HandlerInterceptor: 处理器拦截器 是一个接口,如果需要完成拦截处理需要实现这个接口
5.HandlerExecutionChain: 处理器执行链
6.HandlerAdapter: 处理器适配器 Handler执行业务方法之前,需要一系列的操作,包括表单的校验，数据类型转换等。
7.ModelAndView: 装载了模型数据和视图信息,作为Handler的处理结果,返回给Dispatcher。
8.ViewResolver: 视图解析器 Dispatcher通过它将逻辑视图解析为物理视图,最终将渲染结果返回给客户端。

Spring MVC工作流程

我们来看看Spring MVC工作流程图

1.用户向客户端发送请求,这个请求会先被前端控制器拦截。
2.前端控制器接收到用户请求后会调用HandlerMapping处理器映射器,可以知道该请求由哪个controller处理
3.DispatcherServlet调用HandlerAdapter处理器适配器，告诉处理器适配器应该要去执行哪个Controller
4.HandlerAdapter处理器适配器去执行Controller并得到ModelAndView(数据和视图)，并层层返回给DispatcherServlet
5.DispatcherServlet将ModelAndView交给ViewReslover视图解析器解析，然后返回真正的视图。
6.DispatcherServlet将模型数据填充到视图中
7.DispatcherServlet将结果响应给用户

Spring MVC特点

1.清晰的角色分配:控制器(controller),前端控制器(DispatcherServlet),处理器映射器(HandlerMapping)等,每一个角色都由一个专门的对象来实现。
2.可定制的HandlerMapping和ViewResolver:SpringMVC提供更简单的URL映射
3.可以支持各种视图技术,而不仅仅是jsp
4.有大量的接口,面向接口编程。

Spring MVC注解

1.@RequestMapping:Spring MVC通过@RequestMapping注解将URL请求与业务方法进行映射
参数：
(1).value:指定URL的实际地址
(2).method:指定请求类型,一般为get,post,put,delete.加上method之后,只能通过它规定的请求方式去请求
(3).params:指定请求中必须包含请求参数
如果加上params,访问浏览器不加参数,会报错


2.@Controller:在类上定义,将类交给Ioc容器管理,可以接收用户的请求。
3.@PathVariable用于将请求URL中的模板变量映射到功能处理方法的参数上。

4.@RequestParam:将请求参数绑定到你控制器的方法参数上（是springmvc中接收普通参数的注解

假如value=”num” 将http请求中名为num的参数赋值给形参id
required:设置num是否为必填项,true表示必填,false表示不是必填
defaultValue:设置一个默认值,默认为0

这里注意:如果传过来的参数和你命令参数名称一致,可以不加这个注解。

5.@requestBody:将json数据解析为对象

java bean:SpringMVC会根据请求参数名和JavaBean的属性名自动匹配,自动为对象填充属性值,支持级联属性(对象包含对象)
